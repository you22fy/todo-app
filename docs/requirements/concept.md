# コンセプト

## 背景

個人のタスク管理において、単純なTODOリストでは以下の課題が生じる。

- タスクが増えると全体像が見えなくなる
- プロジェクト単位での進捗把握ができない
- タスク間の依存関係（前後関係）を管理できない
- 優先度による分類・フィルタリングができない
- 大きなタスクを細分化して管理できない

本アプリケーションはこれらの課題を解決するため、**階層的なタスク管理**と**依存関係の追跡**を備えた個人用TODOアプリケーションを構築する。

## 目的

- **プロジェクト→タスク→サブタスク** の3階層でタスクを体系的に管理する
- タスク間の**ブロック関係**を明示し、着手可能なタスクを明確にする
- **サブタスクの状態から親タスクのステータスを自動計算**し、手動管理の手間を削減する
- 複数のビュー（優先度別・全件・プロジェクト別）で状況を素早く把握する

## 対象ユーザー

- 本アプリケーションの利用者は**個人（1名）**である
- マルチユーザー機能は不要
- 認証はProxy経由のBasic認証で簡易的に保護する

## 設計方針

| 方針 | 内容 |
|---|---|
| シンプルさ | 複雑で奇抜な実装ではなく、確実に動く実装を優先する |
| UX重視 | 楽観的更新によりUI操作を即座に反映し、待ち時間を排除する |
| テスト可能性 | ビジネスロジックを純粋関数として切り出し、高カバレッジを実現する |
| ローカル完結 | データベースにSQLiteを採用し、外部サービスへの依存を排除する |
| レスポンシブ | スマートフォン・PCの両方で快適に操作できるUIとする |

## スコープ外

- ダークモードの実装
- マルチユーザー対応
- 通知機能（メール・プッシュ通知）
- タスクのドラッグ&ドロップ並び替え
- ファイル添付機能
